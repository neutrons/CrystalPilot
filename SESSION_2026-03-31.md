## Phase 5b — Tab Navigation (`navigate_to_tab`)

**Problem:** The agent could set parameters and read state but had no way to direct the user to a specific UI tab.

**Root cause of `active_tab` type bug:** `ViewState.active_tab` was declared `str = "0"` but the VTabs widget uses integer values (1–6). When `update_in_view` pushed a string back, Vue3's strict equality (`controls.active_tab == 1`) failed silently.

**Fix:**

| File | Change |
|------|--------|
| `view_models/main.py` | `active_tab: str = "0"` → `active_tab: int = 0` |
| `view_models/main.py` | Added `navigate_to_tab(tab_number: int)` method |
| `agent/tools.py` | Added `_TAB_MAP`, `_TAB_NAMES` module-level dicts |
| `agent/tools.py` | Added `nav_fn=None` param to `make_tools`; `navigate_to_tab` tool |
| `agent/agent.py` | Added `nav_fn=None` to `Agent.__init__`; passes to `make_tools` |
| `agent/agent.py` | Added `_handle_navigate_to_tab` handler; routing in validator |
| `agent/agent.py` | Updated system prompt |
| `app/view_models/chat.py` | Added `nav_fn=None` to `ChatViewModel.__init__`; stores as `_nav_fn`; passes to `Agent` |
| `app/mvvm_factory.py` | Passes `nav_fn=main_vm.navigate_to_tab` to `ChatViewModel` |

## Phase 7 — RAG Knowledge Base

**Problem:** The agent could only answer questions using its base LLM knowledge. There was no structured source of truth for instrument-specific parameters, crystallography concepts, or beamline procedures.

**Design decisions:**
- **No new dependencies** — use `scikit-learn` (already `>=1.6.1` in `pyproject.toml`) for TF-IDF + cosine similarity
- **No external services** — fully local, works offline
- **Drop-in extensibility** — any `.md` / `.txt` file placed in `agent/knowledge/` is automatically indexed at agent init
- **Two-pass synthesis** — `retrieve_docs` routes `validator → agent` (new `_validator_routing` conditional edge) so the LLM sees the retrieved passages as a ToolMessage and synthesizes a proper natural-language reply rather than dumping raw text

### New files

**`src/exphub/agent/rag.py`** — `BeamlineKnowledgeBase`

```
BeamlineKnowledgeBase(knowledge_dir)
  ├─ _load_documents()     glob **/*.md + **/*.txt, read all
  ├─ _chunk_text()         150-word windows, 25-word overlap
  ├─ _build_index()        TfidfVectorizer(ngram_range=(1,2), max_features=30k)
  └─ retrieve(query, k=3)  cosine_similarity, score > 1e-6 filter
```

**`src/exphub/agent/knowledge/beamline_guide.md`** (~1500 words)

Sections:
- Single-crystal neutron diffraction overview
- Crystal systems and Bravais lattices
- Point groups, space groups, Laue classes
- TOPAZ (BL-12), CORELLI (BL-9), MANDI (BL-11B) instrument specs
- All data reduction parameters (max_q, tolerance, peak_radius, bkg_inner/outer_radius, predict_peaks, pred_min/max dspacing and wavelength, num_peaks_to_find)
- Angle plan fields (phi, omega, PCharge, wait_for, value, or_time)
- IPTS and EIC Control
- Mantid algorithm descriptions (LoadLiveData, ConvertToMD, FindPeaksMD, IndexPeaks, FindUB, PredictPeaks, IntegrateEllipsoids)
- Troubleshooting guide

### Graph flow for RAG queries

```
User question
    ↓
agent node  →  LLM calls retrieve_docs(query)
    ↓
tools node  →  rag.retrieve() returns passages
    ↓
validator   →  _validator_routing sees ToolMessage(retrieve_docs) → "agent"
    ↓
agent node  →  LLM sees retrieved passages, synthesises natural-language reply
    ↓
__end__
```

## Architecture Summary (final)

```
User ──► ChatViewModel.handle_submit()
              │
              ├─ snapshot_models()          UI state → flat dict    (bridge.py)
              ├─ Agent.invoke()             LangGraph turn
              │     ├─ LLM + 15 tools
              │     ├─ validator node       formats tool results
              │     ├─ _validator_routing   loops back to agent for RAG
              │     └─ BeamlineKnowledgeBase  TF-IDF retrieval
              ├─ apply_agent_config()       flat dict → Pydantic → Trame push
              ├─ MainViewModel.navigate_to_tab()  tab switch (Phase 5b)
              └─ ChatModel snackbar fields  field-change notification (Phase 5c)
```
