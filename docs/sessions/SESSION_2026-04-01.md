# Session 2026-04-01: Agent UX, Parameter Fix, Workflow Knowledge

## Changes Overview

This session addressed three areas: chat panel presentation, agent parameter
setting reliability, and injecting experiment workflow knowledge into the agent.

### 1. Chat Bubble UX (chat_pane.py, md_render.py, chat.py viewmodel)

- **Dialogue-style bubbles**: User messages appear in light grey (`#e0e0e0`)
  right-aligned bubbles; agent replies in light green (`#e8f5e9`) left-aligned
  bubbles.
- **Markdown rendering**: Created `md_render.py` — a lightweight Python
  markdown-to-HTML converter (no external dependency). Handles fenced code
  blocks, inline code, bold, italic, headers, lists, and line breaks.
- **Scrollable overflow**: Assistant bubbles have `max-height: 400px` with
  `overflow-y: auto` for long responses, and `overflow-x: auto` for wide
  code blocks.
- **Template fix**: Replaced dynamic `:class` binding (broken in trame's
  `v_for` scope) with `<template v-for>` containing separate `v_if` divs
  with hardcoded `classes=`, matching the working "Thinking..." pattern.

### 2. Agent Parameter Setting Fix (agent.py, tools.py, utils.py, bridge.py)

**Problem**: Sending "set IPTS number to 12132" in the chat did not update
the IPTS field visible on the left pane.

**Root causes**:
- The LLM had no parameter name context on the first turn, causing it to
  guess field names.
- `coerce_type()` didn't handle `type == "string"`, so int values passed
  through and were rejected by Pydantic/binding validation.
- Two separate `ipts_number` fields existed on different models
  (`ExperimentInfoModel.ipts_number` vs `EICControlModel.IPTS_number`),
  and the agent only updated one.

**Fixes**:
- **Parameter name injection**: Always inject available parameter names into
  the LLM system context (not just in config mode).
- **Fuzzy matching**: Added `resolve_param_name()` with 3-tier resolution:
  exact match, normalized match (case/underscore insensitive), unique
  substring fallback. Used by `agent.py` and all parameter-related tools.
- **String coercion**: Added `if t == "string": return str(value)` in
  `coerce_type()`.
- **Unified IPTS field**: Renamed `EICControlModel.IPTS_number` to
  `ipts_number` to match `ExperimentInfoModel`. Updated all references
  (model, views, commented code). Removed the temporary `_LINKED_FIELDS`
  mirroring hack.
- **Diagnostic prints**: Added `[Bridge]` debug prints in `bridge.py` and
  config diff logging in `chat.py` viewmodel.

### 3. Experiment Workflow Knowledge (system_prompt.md)

Added two new sections to the agent's system prompt:

- **Experiment Workflow table**: 7-phase sequence (Setup, Monitor, Plan,
  Refine, Submit, Observe, Analyse) mapping each phase to its CrystalPilot
  tab and user actions. Enables the agent to proactively guide users through
  the experiment lifecycle.
- **NeuXtalViz (NXV) guidance**: Phase 4 peak-specific strategy optimisation
  workflow — Show Coverage -> NXV optimisation -> export -> import back into
  CrystalPilot. Informational only (no tool calls needed).
