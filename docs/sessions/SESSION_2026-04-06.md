# Session 2026-04-06: Experiment Steering — NeuXtalViz File-Based Integration

## Objective

Implement the file-based integration between CrystalPilot's Experiment Steering
tab and NeuXtalViz (NXV) so that clicking "Show Coverage" exports the current
angle plan as a CSV, launches NXV with it, and auto-reimports the edited plan
when NXV closes.

## Background

The Experiment Steering tab (tab 3) renders `AnglePlanView` and combines two
models: `AnglePlanModel` (strategy table, coverage geometry) and
`EICControlModel` (EIC submission, auto-stop). The desired workflow is:

1. (Optional) Generate initial strategy via "Initialize Strategy"
2. Click "Show Coverage" → launch NXV with current plan → user edits visually
   → NXV auto-saves on close → CrystalPilot reimports
3. (Optional) Further manual edits in the table
4. Submit through EIC
5. Auto-steering monitors live data and stops runs when thresholds are met

Previously, "Show Coverage" simply ran `os.system("~/run-nxv.sh")` — a blocking
call with no data exchange. The NXV fork had a partial `--initialize-planner`
CLI flag but no CSV round-trip capability.

## Approach Evaluated

Three approaches were considered:

- **A: File-based exchange** (chosen) — CSV round-trip, subprocess lifecycle
- **B: Absorb NXV planning logic** — too heavy, duplicates coverage engines
- **C: Hybrid** — CrystalPilot optimizer + NXV for visual editing

Approach A was selected for its simplicity and clean separation of concerns.

## Implementation

### CrystalPilot Changes (2 files)

#### `src/exphub/app/models/angle_plan.py`

Added two methods to `AnglePlanModel`:

- **`export_to_nxv_csv(file_path)`** — Writes `angle_list` to CSV with NXV
  motor column names:
  - `BL12:Mot:goniokm:omega`, `BL12:Mot:goniokm:chi`, `BL12:Mot:goniokm:phi`,
    `comment`
  - Maps internal `omega`/`chi`/`phi` keys to motor names; puts `title` into
    `comment` column

- **`import_from_nxv_csv(file_path)`** — Reads NXV's CSV back into
  `angle_list`, filling EIC defaults (`wait_for="PCharge"`, `value=10`,
  `or_time=""`)

#### `src/exphub/app/view_models/main.py`

Rewrote `show_coverage()`:

1. Exports current `angle_list` to `/tmp/crystalpilot_nxv_plan.csv`
2. Reads UB file path from `self.model.experimentinfo.UBFileName`
3. Builds shell command: `source activate && conda activate nxvnew && python
   NeuXtalViz.py --initialize-planner <UB> --open-plan <csv>`
4. Launches via `subprocess.Popen` (non-blocking)
5. Spawns `_wait_for_nxv_and_reimport()` async task that:
   - Waits for the subprocess in a thread executor
   - On exit, calls `import_from_nxv_csv()` and `update_view()`

Added `import os, subprocess, tempfile` to top-level imports.

### NeuXtalViz Fork Changes (2 files)

#### `src/NeuXtalViz.py` (main entry)

- Added `self._plan_csv_path = None` to `__init__`
- Added `closeEvent()` override: if `_plan_csv_path` is set and `mtd["plan"]`
  exists, calls `self.ep.update_plan()` then `self.ep.model.save_plan(path)`
- Modified `initialize_planner(filename, plan_csv=None)`:
  - Made `filename` (UB) optional — NXV can launch with just `--open-plan`
  - If `plan_csv` provided, calls `model.load_plan_from_csv()` and
    `view.add_settings()` to populate the planner table
  - Stores `plan_csv` path on window for auto-save on close
- Added `--open-plan` argparse argument
- Changed `__main__` block to dispatch on either `--initialize-planner` or
  `--open-plan`

#### `src/NeuXtalViz/models/experiment_planner.py`

Added `load_plan_from_csv(filename)`:
- Reads CSV via `csv.DictReader`
- Dynamically determines angle column names (everything except `comment`/`use`)
- Parses angles to float, collects comments, sets `use=True` for all rows
- Calls existing `create_plan()` to populate `mtd["plan"]` TableWorkspace
- Returns `(settings, comments, use)` tuple for the view

## CSV Format Agreement

CrystalPilot and NXV exchange via this CSV format:

```csv
BL12:Mot:goniokm:omega,BL12:Mot:goniokm:chi,BL12:Mot:goniokm:phi,comment
0.0,135.0,0.0,CrystalPilot
10.0,135.0,45.0,CrystalPilot
```

NXV's `save_plan()` writes the same column structure (minus `use` column which
is filtered out). CrystalPilot's reimport adds back EIC-specific defaults.

## Data Flow Diagram

```
CrystalPilot angle_list
    │
    ▼ export_to_nxv_csv()
/tmp/crystalpilot_nxv_plan.csv
    │
    ▼ subprocess: python NeuXtalViz.py --open-plan <csv>
NXV loads CSV → load_plan_from_csv() → mtd["plan"]
    │
    ▼ user edits strategy visually in NXV
    │
    ▼ user closes NXV → closeEvent → save_plan() → same CSV
/tmp/crystalpilot_nxv_plan.csv (updated)
    │
    ▼ _wait_for_nxv_and_reimport() detects process exit
    ▼ import_from_nxv_csv()
CrystalPilot angle_list (updated) → update_view()
```
